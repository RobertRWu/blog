Description

URL Functions

path

Description

python

This function returns an element for inclusion in urlpatterns. It has four aguments:

  • route

    his argument should be a string or gettext_lazy() that contains a URL pattern.

  • view

    This argument is a view function or the result of for class-based views. It can also be an django.urls.include().

  • kwargs

    This argument allows you to pass additional arguments to the view function or method.

  • name

    Name an URL so we can refer it anywhere, especially in template.

Pattern

  • Use <> to capture values and send it as a keyword argument to the view.
  • The angle brackets may include a converter specification. A converter will limits the characters matched and may also change the type of the variable passed to the view.
  • No need to add ‘/‘ in the beginning.

Path Converter

There are five converters:

  • str

    Match string composed without slash ‘/‘.

  • int

    Match a string of decimal digits.

  • slug

    Match string composed only by letters, digits, ‘-‘ and ‘_’.

  • uuid

    Match string like “124saf-g1214-saf2”.

  • path

    Match every string.

It will match URL like /articles/2020/03/test/ and convert “2020” and “03” into int and then they are passed as arguments year and month to the view function article().

This function returns an element for inclusion in urlpatterns. It uses regular expression for route argument.

Description

If we have multiple apps in a project, it will be hard to manage all urls of these apps in a single URLconf module, so we can create several other URLconf modules and use include() function to include other URLconf modules in this place.
