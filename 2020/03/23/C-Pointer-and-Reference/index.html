Pointer

Wild Pointers

  • Uninitialized pointers.
  • The memory that the pointer to is released and forget to set NULL to the pointer.
  • The pointer access data out of bound.

Reference

Properties

  • Reference as parameter won’t occupy memory.
  • Defined reference’s size is the same as pointer.
  • Defined reference can be used as a const pointer.
  • Reference can’t be NULL.
  • Only const reference can refer the const variable.

Smart Pointers

Introduction

Description

Smart pointer is a wrapper class over a pointer. A smart pointer can automatically call delete, so we don’t need to care about the space that we had allocated.

There are several kinds of smart pointers in C++:

  • auto_ptr
  • unique_ptr
  • shared_ptr
  • weak_ptr

Why We Use Smart Pointers?

  1. When we use malloc/new, we may forget to use free/delete
  2. If there is an error while using malloc/new to allocate memory, we can’t release the memory that is already allocated.

shared_ptr

Properties

  • Several shared_ptr can point to the same object. Thus there is a reference counting that records how many different pointers point to the same object.
  • The group of owners of a pointer become responsible for its deletion when the last one of them releases that ownership.

Define and Initialize

c++

Functions

Member Functions

Name Description
unique() If there is no other pointer share the same object, return true
get() Get the pointer
reset() Reset the object that the pointer points to
use_count() Return the number of shared_ptr that share the same object
swap(p, q) Swap pointers in p and q

Non-member Functions

Name Description
make_shared<T>(args) Return a shared_ptr which points to an object with the value args that is dynamic allocated

make_shared()

std::make_shared makes one memory allocation for both the object and data structure required for reference counting.

c++

reset()

Reset the pointer to make it become a nullptr or points to another object.

c++

Cyclic Dependency

When there are two classes A and B, both have pointers that point to other classes. So, it’s always like A is pointing to B and B is pointing to A. Hence, use_count will never reach zero and they never get deleted.

unique_ptr

Properties

  • Several shared_ptr can not point to the same object.
  • It can’t be copied or assigned.

Define and Initialize

c++

Functions

Member Functions

Name Description
release() Delete the pointed object, make the pointer point to nullptr
reset() Make the pointer point to another object
swap(p, q) Swap pointers in p and q
get() Get the pointer

release

Delete the pointed object, make the pointer point to nullptr. This function will return the pointer it used to store.

weak_ptr

Properties

  • This pointer points to an object that is controlled by a shared_ptr
  • This pointer will not affect shared_ptr‘s reference counting.

Functions

Name Description

auto_ptr

Properties

  • It is deprecated now.
  • It can’t point to an array because it calls delete instead of delete[].
  • It can’t be used in STL containers because it can’t be copied.
