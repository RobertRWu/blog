C++ STL Sequence Containers
Created 2020-03-23| Updated 2020-04-02|Computer ScienceLanguageC++
Word count:1k|Reading time: 6 min
| Post View:|comments:

array

Description

Arrays are fixed-size sequence containers: they hold a specific number of elements ordered in a strict linear sequence.

Feature

  • Fixed size, can’t insert or delete element.
  • Must indicate the size when define an array.

Member Functions

back()

Description

Return the last value.

Parameters

None.

Return Value

The last value.

Time Complexity

$O(1)$

data()

Description

Return the start address of the stored data.

Parameters

None.

Return Value

Pointer to the data contained by the array object.

Time Complexity

$O(1)$

fill()

Description

Set each element to a certain value.

Parameters

  • value

    The value to be assigned

Return Value

None.

Time Complexity

$O(n)$

front()

Description

Return the first value.

Parameters

None.

Return Value

The first value.

Time Complexity

$O(1)$

size()

Description

Get the number of elements.

Parameters

None.

Return Value

unsigned int

The number of elements.

Time Complexity

$O(1)$

Iterator

  • Category

    Contiguous iterator.

Code Example

c++

forward_list

Properties

  • Unidirectional list.
  • Doesn’t have size()

Iterator

Properties

  • Category:

    Forward list iterator.

list

Description

Bidirectional cyclic list.

List Node

Each node has 3 member variables:

  • Pointer to next node.
  • Pointer to previous node.
  • Value

There is an empty node in the list. You can view it as the first node or the last node.

Member Functions

begin()

Description

Return an iterator points to the beginning element.

Implementation

Return the iterator point to the node behind the built-in node of list.

end()

Description

Return an iterator points to position behind the last element.

Implementation

Return the iterator point to the built-in node of list.

size()

Description

Get the number of elements.

Parameters

None.

Return Value

unsigned int

The number of elements.

Time Complexity

$O(1)$

Iterator

Properties

  • Category

    Bidirectional iterator.

Implementation

operator++

Assign the current node’s next node pointer to itself.

operator—

Assign the current node’s previous node pointer to itself.

operator*

Return current node’s value.

vector

Description

Vectors are sequence containers representing arrays that can change in size.

Member Variables

vector has three built-in iterators:

  • start

    Point to the first element of vector.

  • finish

    Point to the last element of vector.

  • end_of_storage

    Point to the end of allocated memory.

Member Functions

back()

Description

Return the last value.

Parameters

None.

Return Value

The last value.

Time Complexity

$O(1)$

capacity()

Description

Get the allocated size of the container, including space hasn’t given to elements.

Parameters

None.

Return Value

unsigned int

The size of the currently allocated storage capacity.

Implementation

Return the distance between end_of_storage and start.

Time Complexity

$O(1)$

data()

Description

Return the start address of the stored data.

Parameters

None.

Return Value

Pointer to the data contained by the object.

Time Complexity

$O(1)$

front()

Description

Return the first value.

Parameters

None.

Return Value

The first value.

Time Complexity

$O(1)$

push_back()

Description

Add element at the end.

Parameters

  • val

    Value to be copied (or moved) to the new element.

Return Value

None.

Implementation

Different situations:

  • There is free space:

    Push the new element directly.

  • The space of current vector is not enough:

    • Reallocate a new space which is 2 times or 1.5 times bigger than the old one.
    • Then move elements of the old vector to the new vector one by one.
    • Destroy the old one.
    • Finally push the new element.

  • The space of the machine is not enough:

    Throw an error ``

Time Complexity

  • When there is enough space:

    $O(1)$

  • When an reallocation is needed:

    $O(n)$

size()

Description

Get the number of elements.

Parameters

None.

Return Value

unsigned int

The number of elements.

Implementation

Use finish and start.

Time Complexity

$O(1)$

Iterator

Use the regular pointer as the iterator. So when goes into the iterator traits, it will become random_access_iterator.

deque

Properties

  • deque is constituted by several segments of memory space that have the same size. Each segment is a contiguous memory space but these segments are not contiguous.
  • deque uses a vector to record the start address of each segment. We call this vector “map”. These addresses are put in the center of vector so its helpful when expand the head or the tail.

Member Variables

  • start

    deque iterator.

    Point to the first element of deque.

  • finish

    deque iterator.

    Point to the last element of deque.

  • map

    The vector that record the start address of each segment.

  • map_size

    The number of segments.

Member Functions

back()

Parameters

None.

Return Value

The last element.

front()

Parameters

None.

Return Value

The first element.

insert()

Parameters

  • position

    deque_iterator

    The iterator point to the position where you want to insert the element.

  • value

    The value of element.

Return Value

None.

Implementation

  • If the inserting position is near the beginning, then move elements in front of the inserting position forward to the front end.

  • If the inserting position is near the end, then move elements behind the inserting position backward to the tail end.

push_front()

Implementation

  • If the first segment does not have slots to push the new element, deque will allocate a new segment in front of the first and insert the new element from end to start.

push_back()

Implementation

  • If the last segment does not have slots to push new the element, deque will allocate a new segment behind the last and insert the new element from start to end.

Iterator

Properties

  • Category

    Random access iterator.

Member Variables

There are 4 member variables for the iterator of deque:

  • cur

    Pointer to current element of current segment.

  • first

    Pointer to the first element of current segment.

  • last

    Pointer to the last element of current segment.

  • node

    Pointer to the current segment in the map.

Operators Overloading

operator*

Return *cur.

operator-

If x - y

The distance between two iterators equals to (the sum of following items):

  • Calculate the number of segments between x and y and get the number of elements in these segments.
  • The distance between x.cur and x.last.
  • The distance between y.cur and y.first.

operator++

++x

  • cur++
  • If cur == last, node++, cur = first

x++

  • Invoke ++x

stack

Properties

  • stack is an adaptor based on deque (list and vector are also optional). You should know that stack does not inherit from deque but has a member that is deque.

queue

Properties

  • queue is an adaptor based on deque (list is also optional). You should know that queue does not inherit from deque but has a member that is deque.
