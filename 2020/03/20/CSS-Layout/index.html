Basic Conception

Containing Block

Description

Containing block is the nearest ancestor block element of current element.

Normal Flow

Description

Website is a multi-level structure and only the top level is displayed. We call the lowest level “normal flow”. Every element that we create is in the normal flow by default.

Properties

In the Normal Flow

  • Block Element

    • Block element will occupy a line exclusively.
    • The default width equals to its parent’s width.
    • The default height equals to its children’s height.

  • Inline Element

    • Inline element will not occupy a line exclusively.
    • If a line can’t contain all inline elements, then the exceeding element will occupy a new line.
    • Inline elements do not support width and height.
  • Inline-block Element
    • A block element and do not occupy a line exclusively.

Not in the Normal Flow

We can call element not in the normal flow “floated element”.

  • Elements in the normal flow can overlap with floated elements. But text will not be overlapped, instead, text will wrap the floated element.
  • The floated element will be placed under the bottom of normal flow.
  • All floated elements will display at top left by default and can be overlapped with elements in the normal flow.
  • Floated elements are displayed by order, which means they can’t be overlapped with each other.
  • Floated element will not exceed its parent’s area.
  • Floated element is arranged horizontally.
  • Floated element is like inline-block element (Both block and inline).

CSS Box Model

Description

Each element in CSS can be regarded as a box model. This model includes four parts:

  • Margin

    Have no color.

  • Border

    Its color is affected by the background color.

  • Padding

    Its color is affected by the background color.

  • Content

Attributes about Box Model

height and width

Properties:

  • If the parent’s height is smaller than its children’s height, the children will exceed its parent’s area.
  • Inline elements do not support width and height.

overflow

Set how does the parent element deal with the situation when its children exceed its area.

Here are some possible values:

  • visible

    Default value, allow the overflow .

  • scroll

    Add a vertical scroll bar and a horizontal scroll bar to see the overflow content.

  • hidden

    The overflow part can’t be seen.

  • auto

    If there is a overflow, there will be a scroll bar.

margin

Properties:

  • margin-bottom and margin-right will affect other box’s position
  • margin-top and margin-left will affect its own position
  • margin for inline elements will not affect layout in vertical direction, which means other elements below will not move down.

Vertical Margin:

If two boxes “a” and “b” are adjacent, for example, a is above b. No matter they are siblings or parent and child, their vertical margin will choose the maximum value of a’s margin-bottom and b’s margin-top instead of the sum of them.

For example:

css

The distance between a and b will be 200px instead of 300px.

padding

Properties:

  • paddings for inline elements will not affect layout in vertical direction, which means other elements below will not move down.

border

Properties:

  • borders for inline elements will not affect layout in vertical direction, which means other elements below will not move down.

display

Set the element’s display type.

Here are some possible values:

  • inline

    Set as an inline element.

  • block

    Set as a block element.

  • inline-block

    Set as a block element and do not occupy a line exclusively.

  • table

    Set as a table

  • none

    Don’t display in the website. The element doesn’t occupy the space.

visibility

Set if display or not.

Here are some possible values:

  • visible

    Visible.

  • hidden

    Make the element transparent. You can’t see it but it still occupy the space.

box-sizing

The box-sizing sets how the total width and height of an box is calculated.

Here are some possible values:

  • content-box

    If you set an element’s width to 100 pixels, then the element’s content box will be 100 pixels wide, and the width of any border or padding will be added to the final rendered width, making the element wider than 100px.

  • border-box

    If you set an element’s width to 100 pixels, that 100 pixels will include any border or padding you added, and the content box will shrink to absorb that extra width. This typically makes it much easier to size elements.

Position

Float

Description

We can make an element move left or right relative to its parent element.

Value

  • none

    Default, do not float.

  • left

    Float to left. Make the element float from the normal flow.

  • right

    Float to right. Make the element float from the normal flow.

Properties

  • When float is set, the element’s box will not occupy the whole horizontal direction.

position

Description

This sets how an element is positioned in a document. The top, bottom, left and right properties determine the final location of positioned elements.

Values

  • static

    The element is positioned according to the normal flow of the document. The top, bottom, left, right and z-index properties have no effect. This is the default value.

  • relative

    The element is positioned according to the normal flow of the document, and then offset relative to itself based on the values of top, right, bottom, and left.

  • absolute

    • Element with absolute will be floated element.
    • Element’s containing block will be the nearest ancestor block element with the position open.
    • Element will be positioned relative to its containing block.

  • fixed

    • fixed is a special case of absolute
    • fixed element always positions relative to the viewport.
    • fixed element won’t scroll with the scroll bar.

  • sticky

    • sticky is a special case of relative
    • sticky element will become fixed when scroll it to a certain position.

Priority

left > right (left, margin-left, padding-left), top > bottom

left > margin-left

top > margin-top

z-index

Description

z-index sets the z-order of a positioned element and its descendants or flex items. Overlapping elements with a larger z-index cover those with a smaller one.

Flex Box

Description

Use display: flex to set an element as a flex box. All of its children element will be flex items.

A flex box has two axes, the main axis and the cross axis. The main axis is defined by the flex-direction attribute, And the cross axis runs perpendicular to it.

Properties

  • Flex items will automatically adjust their width when the sum of children’s width is bigger than their parent’s width.

Attributes for Flex Box Parents

justify-content

Set the arrangement format for children elements on the main axis.

Here are some possible values:

  • flex-start:

    Default. Make children elements arrange from its own initial position.

  • flex-end

    Make children elements arrange from its own end position. But the order is not changed.

  • flex-center

    Make children elements arrange in the center.

  • space-between

    Make the first child align to the start position and the last child align to the end. The space between each child is the same.

  • space-around

    Allocate redundant space around both sides of each child.

align-items

Set the arrangement format for children elements on the cross axis.

Here are some possible values:

  • flex-start:

    Default. Make children elements arrange from its own initial position.

  • flex-end

    Make children elements arrange from its own end position. But the order is not changed.

  • flex-center

    Make children elements arrange in the center.

  • stretch

    Default value. Stretch items on the cross axis, make them fill up the cross axis when the width of child element is not defined.

  • baseline

    Arrange according to the baseline of English letters.

flex-wrap

Determine whether start a new line when the width is not enough to contain all flex items.

Here are some possible values:

  • nowrap

    Default. No new line

  • wrap

    Yes.

  • reverse

    Arrange from bottom to top.

flex-direction

Set the main axis.

Here are some possible values:

  • row

    Arrange horizontally from left to right.

  • reverse-row

    Arrange horizontally from right to left.

  • column

    Arrange vertically from top to bottom.

  • reverse-column

    Arrange vertically from bottom to top.

flex-flow

A combination of flex-wrap and flex-direction.

Attributes for Flex Box Children

flex-grow

The flex-grow property can be used to distribute space in proportion. You should use integer to set this attribute. The default value is 0, which means this element will not take up additional space.

Here is how it works:

The proportion of an item = its flex-grow ÷ the sum of all items’ flex-grow.

For example, if there are 4 flex items. The first item’s flex-grow is 2 and others’ are 1. Then the space proportion of the first flex item will be 2 ÷ (2 + 1 + 1 + 1) = 0.4.

You should know flex items will not shrink because of this attribute.

flex-shrink

If we do not have enough space in the container to lay out our items and flex-shrink is set to a positive integer the item can become smaller than the flex-basis. As with flex-grow different values can be assigned in order to cause one item to shrink faster than others — an item with a higher value set for flex-shrink will shrink faster than its siblings that have lower values.

For example, if we have 3 flex items with flex-shrink 2, 1, 1. Each item’s width is 200px and the size of flex box is 500px. Then the size of these items will shrink to 150px ,175px and 175px. You can see that the shrinking size is 50, 25, 25, which is 2:1:1.

The default value is 1. If set to 0, then this element will not shrink. There will be an overflow when the size of flex box can’t contain these kind of flex items.
