History

history

Description

Display commands that has ben executed recently.

Usage

shell

Options

  • -c

    Clear all history in the buffer.

  • -w

    Write history in the buffer to history recording files.

Shortcut Keys

History

  • !n

    Execute the nth history command.

  • !!

    Execute the last command.

  • !string

    Execute the last command that starts with string.

Completion

  • Tab

    Automatically complete the command.

Ctrl

  • ctrl+a

    Move the cursor to the beginning.

  • ctrl+c

    Abort current command.

  • ctrl+d

    Quit current terminal.

  • ctrl+l

    Clear the window.

  • ctrl+r

    Search in history.

  • ctrl+u

    Cut the command before cursor.

  • ctrl+y

    Paste.

Alias

alias

Description

Set an alias to a command.

Usage

shell

unalias

Description

Cancel an alias to a command.

Usage

shell

Symbols

I/O Redirection

Standard Output Redirection

  • command > file

    Cover mode. Output the result of command to specified file or device.

  • command >> file

    Append mode.

Standard Error Redirection

  • command 2> file

    Cover mode.

  • command 2>> file

    Append mode.

Standard Output and Error Redirection

  • command > file 2>&1

    Cover mode. Output command result and error to the file.

  • command >> file 2>&1

    Append mode.

  • command &>file

    Cover mode.

  • command &>>file

    Append mode.

Standard Input Redirection

  • wc < file_name

Logical Operator for Commands

Sequence Execution

Symbol Usage Description
; Command1 ; Command2 Execute 1 and 2 in order.
&& Command1 && Command2 If 1 failed, 2 won’t be executed.
` ` Command1 \ \ Command2 If 1 suceeded, 2 won’t be executed.

Common Examples

shell

Pipe Symbol

Description

Use command1’s correct exection output as the input of command2. If command1 failed, command2 won’t execute.

Usage

shell

Common Examples

shell

Wildcards

Symbol Description
? Matches any single character.
* Matches zero or more charcters.
[] Match a single character in a specified range.
[^] Match a single character that is not in the specified range.

Other

Symbol Description
'' All specific symbols in the '' have no special meanings, they are just characters.
"" All specific symbols except $, \ and ` have no special meanings.
`` Content in the `` are system commands. Bash will excute it first.
$() The same as ``
# Comment
$ Get the value of variable. For example, $var.
${} Get the value of variable. Used for concatenation.
$(()) Used for integer calculation.

Variables

Categories

  • User Defined Variables

  • Environment Variables

  • Positional Parameters

    Used to transfer arguments or data to script. The name this kind variable can’t be customized.

  • Pre-defined Variables

    Variables defined by Bash.

User Defined Variables

Properties

  • Default type is string.
  • When assign a value to a variable, there should not be spaces between value and = or variable and =.
  • If we want to assign a value with space, we need to use "" or '' to include the value.
  • Only works in the current shell.

Usage

shell

Environment Variables

Properties

  • Works in current shell and child shell. But if written in the corresponding configuration file, it will work in all shell.

Usage

shell

Positional Parameters

Used to pass arguments.

Symbol Description
$n n is number, $0 represents the command itself, $1-$9 represents arguments 1-9. ${10} means the tenth arguments and so does 11, 12….
$* Represents all arguments in the command line. They will be seen as a whole.
$@ Represents all arguments in the command line.
$# Represents the number of arguments.

Pre-defined Variables

Symbol Description
$? The execution state of last command. If executed correctly, this variable will be 0.
$$$$ Current PID
$! The last PID running in the background.
