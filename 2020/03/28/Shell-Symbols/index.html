    I/O

    I/O Redirection

    Standard Output Redirection

    • command > file

      Cover mode. Output the result of command to specified file or device.

    • command >> file

      Append mode.

    Standard Error Redirection

    • command 2> file

      Cover mode.

    • command 2>> file

      Append mode.

    Standard Output and Error Redirection

    • command > file 2>&1

      Cover mode. Output command result and error to the file.

    • command >> file 2>&1

      Append mode.

    • command &>file

      Cover mode.

    • command &>>file

      Append mode.

    Standard Input Redirection

    • wc < file_name

    Pipe Symbol

    Description

    Use command1’s correct execution output as the input of command2. If command1 failed, command2 won’t execute.

    Logical Operator for Commands

    Sequence Execution

    Symbol Usage Description
    ; Command1 ; Command2 Execute 1 and 2 in order.
    && Command1 && Command2 If 1 failed, 2 won’t be executed.
    ` ` Command1 \ \ Command2 If 1 succeeded, 2 won’t be executed.

    Common Examples

    shell

    Wildcards

    Symbol Description
    ? Matches any single character.
    * Matches zero or more characters.
    [] Match a single character in a specified range.
    [^] Match a single character that is not in the specified range.

    Parameter Expansion

    Symbol Description
    ${paramter:-word} If the parameter is not set or is null, replace the parameter with word (This won’t replace the value of paramter, only works in this command)
    ${paramter:+word} If the parameter is set and is not null, replace the parameter with word (This won’t replace the value of paramter, only works in this command)
    ${paramter:=word} If the parameter is set and is not null, replace the parameter with word (This will replace the value of paramter)
    ${paramter:?word} If the parameter is not set or is null, output word to stderr (This won’t replace the value of paramter, only works in this command)
    ${paramter:offset} Expand to the substring of parameter that starts from the offset.
    ${paramter:offset:length} Expand to the substring of parameter that starts from the offset and with length as length.

    Other

    Symbol Description
    '' All specific symbols in the '' have no special meanings, they are just characters.
    "" All specific symbols except $, \ and ` have no special meanings.
    `` Content in the `` are system commands. Bash will execute it first.
    $() The same as ``
    # Comment
    $ Get the value of variable. For example, $var.
    ${} Get the value of variable. Used for concatenation.
    $(()) Used for integer calculation.
