Basic Conceptions

Binding

  • Binding:

    Connecting a function call to a function.

  • Early binding:

    Binding a function while compiling which is static binding. For example, a subclass object invokes a function already implemented in the base class.

  • Late binding:

    Binding a function while running, which is dynamic binding. For example, a base class pointer that points to a subclass object and invokes a function implemented in this subclass.

Coupling

The degree to which software components depend on each other.

Singleton Pattern

Description

Make sure that a class has only one instance in a software system, while providing a global access point to this instance.

Implementation

  • Define constructor and copy constructor as private.
  • Define a function for outside to get the instance of class.
  • Define a pointer to the distance.
  • We should implement thread safety.

Factory Method Pattern

Description

When we need to create an instance (new), we can only create non-abstract object which will bring tight coupling (depend on a concrete class). In order to solve this problem, we can use factory method pattern.

Provides an interface for creating objects in a superclass, but allows subclasses to alter the type of objects that will be created.

Bridge Pattern

Description

Lets you split a large class or a set of closely related classes into two separate hierarchies—abstraction and implementation—which can be developed independently of each other.

Decorator Pattern

Description

This pattern lets you attach new behaviors to objects by placing these objects inside special wrapper objects that contain the behaviors instead of add behaviors to the class.

Properties

  • This pattern realizes the ability to achieve additional function dynamically.

Implementation

Template Method Pattern

Description

The library developer define the algorithm and writes the base class then leaves some specific steps to be achieved in subclasses for application developers without changing its structure.

Properties

  • Functions left for subclasses to achieve should be virtual functions.
  • The destructor should be virtual.
  • This changes early binding to late binding.

Implementation

  • We can use a list to contain each observer and use iterator to notify them one by one.

Observant Pattern

Description

Lets you define a subscription mechanism to notify multiple objects about any events that happen to the object they’re observing.

Properties

  • We can modify target or observers independently, which means if we need to modify observers we don’t need to modify the target.
  • When the target sends notification, it don’t need to specify observers.

Strategy Pattern

Description

Define a family of algorithms, put each of them into a separate class, and make their objects interchangeable.

Properties

  • Make object be able to select among algorithms while running.
  • This pattern provides an alternative solution when plenty of conditional statements are needed. You can expand the code instead of modifying previous code by using this pattern.
  • This pattern can decouple components.
